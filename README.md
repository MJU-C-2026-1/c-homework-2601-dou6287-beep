## 1. 시나리오 제목
* 드럼 연습 기록 및 분석 프로그램

## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 드럼 연습 기록을 입력받아 성과를 분석하고 목표 달성도를 확인하는 <b>'드럼 연습 기록 및 분석 시스템'</b>입니다.
* 사용자는 당일 진행한 주요 연습 종류(루디먼트, 곡, 프리)와 총 연습 시간, 목표 템포(BPM), 당일 도달한 최고 템포(BPM) 등의 필수 데이터를 입력합니다. 시스템은 입력된 데이터를 바탕으로 연산을 수행하여 사용자가 도달하고자 하는 목표 BPM과 당일 달성한 최고 BPM의 차이를 계산하여 목표까지 남은 수치를 정확하게 도출해 냅니다. 또한 목표 BPM과 당일 달성한 최고 BPM의 결과를 반영하여 당일 목표 달성률을 소수점 2자리까지 계산하고 도출합니다. 최종적으로 연산된 분석 결과는 가독성 좋게 정렬되어 콘솔에 출력되며, 사용자는 이를 통해 자신의 템포 향상도와 목표 달성률을 매일 객관적으로 점검할 수 있습니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 연습 종류 입력
2. 연습 시간 입력
3. 목표 BPM 입력
4. 당일 도달한 최고 BPM 입력
5. 목표 BPM까지 남은 수치 계산
6. 목표 달성률 계산

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 연습 종류, 총 연습 시간, 목표 BPM, 당일 도달한 최고BPM을 입력받아 목표까지 남은 BPM 수치와 목표 달성률을 계산하여 출력합니다.
  * 데이터의 특성에 맞게 연습 종류(문자), 연습 시간(실수), 템포(정수)를 입력받는 화면을 구현했습니다.
  * 사칙연산을 활용해 '남은 목표 BPM'과 '목표 달성률'을 계산하여 출력했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "관심사인 '드럼'과 이와 관련된 '드럼 연습과 기록'에 대한것 주제로 개발할 C언어 콘솔 프로그램 시나리오를 4단계 구조(입출력-조건문-반복문-배열/함수)에 맞춰 구체적인 구현 로드맵 기획 요청"
    * **적용 내용:** '연습 종류 및 시간 입력(1차)-목표 템포(BPM) 및 당일 도달한 최고 템포(BPM)입력(2차)-목표 템포까지 남은 BPM 수치 계산(3차)-현재 목표 달성률 계산(4차)'으로 이어지는 논리적인 시스템 확장 로드맵과 예상 메뉴를 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "순차 구조, 변수 5개 이상, 자료형 3개 이상, 사칙연산 1회 이상 등 여러 제한 조건을 만족하는 드럼 연습 기록 데이터 구조 모델링 논의"
    * **적용 내용:** 실제 드럼 연습에 필요한 유의미한 데이터(연습 타입, 시간, 템포(BPM))등의 변수를 구조화하여 순차 구조의 완성도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 여러 개의 출력 결과가 한 줄에 붙거나 삐뚤어져서 나와 분석 결과를 한눈에 알아보기 힘든 현상이 발생함.
     * **해결:** printf 출력문 내에 제어 문자(\n 줄바꿈, \t 탭)를 적절한 간격으로 배치하여 가지런히 정렬되도록 함.
  2. **문제 2:** 프로그램에 오류가 발생함.
     * **해결:** scanf함수에서 주소 연산자(&)를 누락한 것을 발견하고 scanf("%d", practice_hours);로 작성했던 부분을 scanf("%d", &practice_hours);와 같이 수정하여, 입력받은 데이터가 변수의 메모리 주소에 정상적으로 저장되도록 고쳐 오류를 해결함.
